## What this is

A GeoGuessr-style geography guessing game set entirely in Goa, India. A player sees a 360° street-level panorama, drops a pin on a label-free map of Goa, and scores points by proximity. Five rounds per game. 100% free to run: static site, no backend, no accounts.

**Audience:** friends & family, shared via WhatsApp links.
**Competition mechanic:** Wordle-style shareable result text carrying a challenge link; everyone who opens the link plays the same five locations.

## Stack

| Concern | Choice |
|---|---|
| UI framework | Svelte 5 (runes) + TypeScript + Vite |
| Styling | Tailwind CSS |
| 360° viewer | mapillary-js v4 (free Mapillary client token, baked into bundle — public by design) |
| Guess map | Leaflet + CartoDB Positron **No Labels** tiles (anti-cheat) |
| Distance | Client-side haversine (pure TS) |
| Tests | Vitest (pure-logic modules only) |

**Structural rule:** game logic lives in plain TS modules; components only render. The two library wrappers (`PanoViewer`, `GuessMap`) are the only files that touch mapillary-js/Leaflet.

## Game flow

Phases: `picking → playing → scored → summary` (per game; `playing → scored` cycles 5×).

1. On load, seeded shuffle deals **5 rounds + 2 backups** from the location pool.
2. Each round: panorama shown (look-around only — Mapillary `sequence`, `direction`, `tag`, `popup`, `cover` components all disabled (`direction`'s arrows are click-to-navigate, so it must be off)), player pans/zooms, drops pin on Leaflet map, submits.
3. Scored view: distance line drawn between guess and answer, round points shown, map fit to both markers.
4. After round 5: summary screen with total and share button.

**Movement:** none. Players look around from a fixed point. Fairest for score comparison, fastest rounds.

### Round reveal (the payoff moment)

When the player submits a guess, the map must show them *how far off they were* — this is the core feedback loop of the game:

- A dashed polyline is drawn from the guess pin to the actual location.
- The actual location appears as a distinct marker (green divIcon, visually different from the guess pin).
- The map animates a `fitBounds` to frame both markers with padding, so the miss distance is *seen*, not just read.
- The footer shows the distance prominently — formatted as meters under 1 km ("340 m away"), kilometers otherwise ("4.2 km away") — alongside the points earned.
- The summary screen repeats per-round distances next to each location name, so the player can relive the whole game.

## Challenge links & seeding

- Bare URL → generate random seed → immediately rewrite URL to `?c=<code>` (history.replaceState), so the game in progress is always shareable.
- `?c=<code>` → decode → same shuffle → identical 5 rounds + 2 backups, same order.
- The code encodes `(seed, poolVersion)`. `locations.json` is **append-only**; each entry records the pool version it joined in. A challenge shuffles only over entries with `version ≤ code.poolVersion`, so old links replay identically after the pool grows.
- Invalid/missing code → silent fallback to fresh random seed.

## Scoring

- Max 5,000/round; 25,000/game.
- Full points at ≤ 25 m. Zero at ≥ 15 km. In between: `5000 × (1 − d/15000)²` (quadratic falloff).
- Haversine on WGS-84 sphere (R = 6371 km) — accurate to well under pin-drop precision at Goa scale.

## Share & competition

All competition rides on the share link — no backend. Four pieces:

### 1. Emoji round bar

Each round's distance becomes a spoiler-free emoji: 🎯 ≤ 100 m · 🟢 < 1 km · 🟡 < 5 km · 🔴 ≥ 5 km. Five emoji tell the story of a game at a glance.

### 2. Goan rank titles

Total score maps to a flavor title, shown on the summary screen and in the share text:

| Score | Title |
|---|---|
| 0 – 6,000 | Confused Tourist |
| 6,001 – 12,000 | Beach Regular |
| 12,001 – 18,000 | Susegad Local |
| 18,001 – 23,000 | Poder of the Village |
| 23,001 – 25,000 | True Goenkar |

(Titles are a constant table in `share.ts`; tweak freely.)

### 3. Head-to-head via link

The share URL carries the sharer's results: `?c=<code>&r=<encoded>&n=<name>`.

- `r`: per-round scores, compactly encoded (5 × 0–5000 fits comfortably in base36). Decoded → the challenger's full game.
- `n`: optional display name, URL-encoded, capped at 20 chars, sanitized to plain text on render.
- A friend opening the link sees a challenge banner ("**Kapil** scored 18,240 — beat them"), after each round's reveal a per-round comparison ("You +3,900 · Kapil +4,120"), a running head-to-head total in the HUD, and a final comparison screen with per-round bars and a winner banner (🏆 / 🤝 on tie).
- The friend's own share then carries *their* results — challenges chain naturally.
- Malformed `r`/`n` are ignored; the game degrades to a plain challenge link.

### 4. Share output

One tap on the summary screen:

- **Mobile** (`navigator.share` with file support): shares a canvas-rendered PNG scorecard — dark card with score, rank title, emoji bar, per-round distances — plus the text below.
- **Desktop / fallback:** copies the text to clipboard.

```
Backyard: Goa 🏖️ — Susegad Local
🎯🟢🔴🟢🟡 18,240 / 25,000
Beat me: <URL>?c=<code>&r=<encoded>&n=Kapil
```

Card rendering lives in `src/lib/card.ts` (offscreen canvas, no DOM screenshot libraries); result encoding/decoding and text building in `src/lib/share.ts`.

## Error handling

| Failure | Behavior |
|---|---|
| Panorama fails to load (deleted image, network) | Swap in next backup location. Deterministic order, so challenge peers substitute identically. |
| All 7 locations fail | "Can't reach street view" screen with retry button. |
| Invalid challenge code | Fresh random seed, no error surfaced. |
| Missing `VITE_MAPILLARY_TOKEN` | Build fails with a clear message (checked in vite config). |
| Leaflet default-icon asset breakage under Vite | Avoided entirely: `divIcon` for all markers. |
| WebGL context leak | `PanoViewer` calls `viewer.remove()` in `onDestroy`; `GuessMap` likewise `map.remove()`. |

## Curation workflow

1. Curator browses Mapillary Explorer over Goa, pastes image IDs into `scripts/spots.txt`.
2. `npm run curate`:
   - Fetches each image's `computed_geometry` and `is_pano` from the Mapillary Graph API (coordinates are **derived from the image ID, never hand-typed**).
   - Rejects: non-pano images, duplicate IDs, coordinates outside a Goa bounding box (approx. 14.85–15.85 N, 73.65–74.35 E).
   - Appends valid entries to `locations.json`, stamped with the current pool version.
3. Launch target: 15–20 spots. Repo ships with a small starter set of verified IDs to replace/extend.

`name` field is stored for the curator's reference and shown only on the **scored/summary** screens (never during play — it would leak the answer).

## Attribution & terms

HUD permanently shows: Mapillary (imagery), © OpenStreetMap contributors, © CARTO (tiles). Required by all three providers' terms.

## Testing

- `score.ts`: haversine against known city-pair distances; points curve at 0 m / 25 m / 15 km / beyond; monotonicity.
- `seed.ts`: same code → same deal (rounds + backups); different codes diverge; pool-version pinning holds after appending locations.
- `game.svelte.ts`: phase transitions, round advance, backup substitution, score accumulation, restart.
- `share.ts`: result encoding round-trips (5 scores → code → same 5 scores); malformed input → null; emoji bar thresholds; rank title bands at boundaries; name sanitization.
- No browser E2E. Library wrappers and canvas card verified by manual smoke test.

## Out of scope (YAGNI)

Accounts, server leaderboards, timers, daily challenge, movement/step limits, multiple regions, anti-cheat beyond no-labels tiles (image IDs are visible in the network tab; accepted for this audience).
